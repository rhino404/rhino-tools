# Blog Pipeline

Local authoring and compile pipeline for the Ryno Tools blog.

**This directory is tracked in git but does NOT deploy.**
The GitHub Actions workflow uploads only `./src` — so only the compiled
`src/blog/` output reaches production.

## Quick start

```bash
# 1. Ideate: rank under-covered topics from the datasets
node blog-pipeline/ideate.mjs
# → writes blog-pipeline/queue.json

# 2. Author a draft (or use the /blog skill via Claude)
# → writes blog-pipeline/drafts/<slug>.md
# Set status: ready when the draft is complete.

# 3. Validate
node blog-pipeline/validate.mjs
# → must be green (zero errors) before building

# 4. Build
node blog-pipeline/build.mjs
# → compiles to src/blog/<slug>/index.html, updates index, feed, sitemap

# 5. Preview
cd src && python3 -m http.server 8080
# → open http://localhost:8080/blog/

# 6. Ask the user to review and authorize the merge to main
```

## Directory layout

```
blog-pipeline/
  schema/
    post.schema.json      Front-matter contract for every draft
  templates/
    post.html             Post page HTML template (placeholders: {{POST_*}})
    index.html            Blog index template ({{BLOG_FEATURED_HTML}}, {{BLOG_CARDS_HTML}})
    card.html             Card snippet template ({{CARD_*}})
  drafts/
    <slug>.md             Markdown drafts with YAML front-matter
  posts.json              Generated manifest (source of truth for index/RSS/related)
  ideate.mjs              Topic ideation script → queue.json
  build.mjs               Compiler: drafts → src/blog/
  validate.mjs            Validator: schema + copyright heuristics
  queue.json              Ranked topic ideas (generated by ideate.mjs)
  README.md               This file
```

## Draft format

```markdown
---
title: "Your Post Title"
slug: your-post-slug
description: "~155 char meta description. Used in <meta>, OG, JSON-LD."
track: ham-radio/technician
tags: [voltage, resistance, current]
datePublished: 2026-06-08
readingTime: 6
excerpt: "Short excerpt for blog index cards (~200 chars max)."
sources_json: [{"title":"ARRL Handbook","url":"https://www.arrl.org/arrl-handbook","publisher":"ARRL","accessed":"2026-06-08"}]
status: ready
---

## What is Ohm's Law?

Ohm's Law states that voltage equals current times resistance...
```

### Key rules

- `status: draft` — WIP; skipped by validate + build
- `status: ready` — passes validation; included in build
- `status: published` — already in src/blog/ (set by build.mjs)
- `sources_json` must be valid JSON array; ≥1 source required
- Body headings start at `##` (the post `<h1>` is the title, set by template)
- No fenced code blocks, no Markdown tables (renderer doesn't support them)
- No external images — CSS banners only; copyright-clean

## Author identity

All posts publish under **"Ask Ryno"** — the Ryno Tools editorial/AI persona,
AI-drafted and human-reviewed before every publish. Author JSON-LD links to the
Ryno Tools Organization. A disclosure line appears in every post footer.

## Publish guard

Push to `main` = instant deploy to ryno.tools. **Always merge on the user's
explicit authorization** after local review. The PreToolUse hook in
`.claude/settings.local.json` blocks accidental `git push` without sign-off.

## Extending the pipeline

- **New track**: add to `TRACK_META` in `build.mjs` and `TRACKS` in `ideate.mjs`.
- **Template changes**: edit `templates/post.html` or `templates/index.html`; run
  `node blog-pipeline/build.mjs` to regenerate all posts.
- **CSS**: edit `src/css/blog.css` and bump `?v=` in the template `<link>` tags.
